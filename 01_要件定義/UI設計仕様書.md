# シンセサイザー学習サイト UI設計仕様書 (案)

## 1. デザインコンセプト
**「操作はアナログ、理解はデジタル」**
* **操作エリア:** 実機のような「ツマミ（Knob）」を回すフィジカルな操作感。
* **表示エリア:** 内部処理を可視化した「グラフ」によるリアルタイムなフィードバック。
* **目的:** ツマミを回した瞬間にグラフが変形することで、音響合成のブラックボックスを透明化する。

## 2. 全体レイアウト
画面を上下2分割、および左から右への信号フロー（Signal Flow）で構成。

| エリア | 配置 | 内容 | 役割 |
| :--- | :--- | :--- | :--- |
| **Visualizer** | 上段 | オシロスコープ、スペクトラム、エンベロープグラフ | **「結果」**の表示。音の形や変化を可視化。 |
| **Control Panel** | 下段 | ツマミ、スイッチ、鍵盤 | **「操作」**の入力。実機の操作感をシミュレート。 |
| **Signal Flow** | 左→右 | OSC → FILTER → AMP (ENV) → FX | 音の流れる順序通りにモジュールを横並び配置。 |

## 3. 各モジュールのUI詳細

### A. オシレーター (VCO)
「音の素材」を選ぶセクション。

* **下段 (操作):**
    * `Waveform`: 波形選択ボタン（Saw / Square / Sine）。
    * `Pulse Width`: 矩形波選択時のみ有効なツマミ。
* **上段 (表示):**
    * **リアルタイム・オシロスコープ**: 現在鳴っている波形を表示。
    * **連動:** `Pulse Width`ツマミを回すと、波形の幅がグニグニと変化する様子を描画。

### B. フィルター (VCF) - [最重要・視覚効果高]
「音を削る」セクション。

* **下段 (操作):**
    * `Cutoff`: 大きめのメインツマミ。
    * `Resonance`: 小さめの補助ツマミ。
* **上段 (表示):**
    * **背景:** リアルタイムの周波数スペクトラム（FFT）。音が鳴ると波打つ。
    * **前景:** フィルター特性カーブ（ローパスフィルターの線）。
    * **連動:**
        * `Cutoff`操作 → カーブが左右にスライド。
        * `Resonance`操作 → カーブの折れ目が鋭く突き出る。
        * **効果:** カーブ外側のスペクトラムが削り取られて消える様子を視覚的に見せる。

### C. エンベロープ (EG / ADSR)
「時間の形」を作るセクション。

* **下段 (操作):**
    * `A`, `D`, `S`, `R` の4つの並列ツマミ。
* **上段 (表示):**
    * **ADSRグラフ:** ツマミの値に基づいた静的な台形グラフ。
    * **再生カーソル:** 鍵盤を押すと、グラフの線上を**「光る点」**が移動する。
    * **連動:**
        * ツマミを回すとグラフ形状が即座に変形。
        * 操作中のセクション（例：Attack）の線だけをハイライト表示。

### D. LFO (変調)
「動き」を与えるセクション。

* **UI表現:** モジュール間をつなぐ「ケーブル」として表現。
* **連動:** LFOの`Rate`（速さ）や`Depth`（深さ）に合わせて、ケーブル自体が脈動（光のアニメーション）し、信号が送られていることを表現。

## 4. デザインテイスト
**「Modern Technical（モダン・テクニカル）」**

* **配色:** ダークモードベース。黒に近いグレー背景に、発光色（シアン、オレンジ、ライムグリーンなど）でグラフや現在値を表示。
* **質感:** スキュアモーフィズム（リアルな質感）は排除し、フラットで視認性の高いベクターデザイン。
    * *理由:* 「古い機械」への憧れよりも、「仕組みの理解」を優先するため、ノイズとなる装飾は削ぎ落とす。Ableton LiveやBitwig Studioのような機能美を目指す。
